The basics

So i found a video from a conference about making a basic setup for a pwa. A small website that can be used as an app. Also add an install button on it. Looked awesome, but the template was not finished.

Lets see what we can make of this. Just using html, css and javascript. With a little bit of help from webpack.

The video

This was the video, so watch this as an introduction if you like.

The thoughts

Next up is thinking about the rest of the setup. The basic setup is to use simple and native functions that are in html, css and javascript. Only basic stuff, which end in not so basic stuff and an awesome template.

Setting things up

The concept

The concept is that the html is workable in the end in a mobile view, and a desktop view. We can all install an app in a browser on a desktop or on a mobile. So everything should work.

Mobile

On mobile you should have a header, content and an app bar on the bottom. So some display: flex; and position: fixed; will help us with that.

Desktop

On desktop the header can be larger and the menu should fit below the header. Here we can re-arange data using display: grid; to position the nav back on top. Also add some position: sticky; so the navbar will stay in place.

Development

Webpack

Next up, the basic setup by the original author was copied. We installed webpack, a dev server and a workbox plugin. Maybe a little bit more helper classes for webpack. But thats the basics. Now we can run a dev server by running webpack-dev-server or nom run serve from the commandline.

Css

For css we implemented some basic setup using css vars. The setup is split between variables and components we import in the main.css file.

For the responsive components this means the bulk of the options are found in variables.css.

Javascript

In the javascript there is an IntersectionObserver to check which tab on the page should be active. And for the smooth scrolling there is the scrollIntoView api. Both very basic setups as they are in most browsers. Don't know if the IntersectionObserver is ideal for the check on wich tab is active, but it seems to work!

The basic setup for the serviceWorker is still there from the original code. Did upgrade the manifest.json with some unique icons for this page. But we did not add more basic stuff there.

Deployment

Did add some function for deployment. For now just to github pages using the gh-pages package.

And thats it

And that's it. Using html as html can be used. Using javascript as javascript can be used, just going back to basics, and making it more awesome. Now you can check, download, use code as an example here. And build more awesome stuff on the internet. If you have ideas, please feel free to tell them. The may be implemented.
